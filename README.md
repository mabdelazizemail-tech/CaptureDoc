# أداة مطابقة المشتريات والمبيعات

أداة عربية محلية تعمل على المتصفح لأتمتة ملف «مطابقة المشتريات والمبيعات.xlsx»:
استيراد الملف الحالي، سحب الفواتير الحقيقية من منظومة الفواتير الإلكترونية المصرية (ETA)،
مطابقة تلقائية بين سطور الشراء والبيع، ثم تصدير ملف Excel بنفس التنسيق الأصلي تماماً.

## التشغيل

انقر نقراً مزدوجاً على **`start.bat`** — سيفتح المتصفح تلقائياً على `http://127.0.0.1:8077`.

> يتطلب Python 3.10+ (موجود بالفعل على هذا الجهاز). أول تشغيل يثبّت المتطلبات تلقائياً.

## خطوات الاستخدام

1. **الإعدادات**: أدخل بيئة المنظومة (تجريبية/فعلية) و Client ID/Secret الخاصة بشركتك
   (تحصل عليها من بوابة الفواتير الإلكترونية → ERP registration)، ثم «اختبار الاتصال».
2. **استيراد Excel**: من الزر أعلى الشاشة لتحميل الملف الحالي كنقطة بداية.
3. **مزامنة ETA**: اختر فترة الإصدار واتجاه الفواتير (واردة = مشتريات، صادرة = مبيعات)
   — تُجلب كل فاتورة بكل سطورها مع ضريبة القيمة المضافة الفعلية (T1).
4. **المطابقة**: شغّل «المطابقة التلقائية» (تعتمد على الصنف والكمية وتقارب التواريخ
   وتتسامح مع اختلافات الإملاء العربي مثل الفشاوي/الفيشاوى)، واعتمد المقترحات أو اربط يدوياً.
5. **تصدير Excel**: يولد ملفاً بنفس أعمدة وصيغ الملف الأصلي (SUBTOTAL ،J2=Q4-H4 ،R2=R4-I4).

## ملاحظات

- إشعارات الدائن من ETA تُستورد بكميات وضرائب **سالبة** وتظهر بشارة «اشعار دائن»،
  وتُكتب في عمود «الاجمالي4» عند التصدير كما في الملف الأصلي.
- المزامنة تحترم حد المنظومة (طلب كل ثانيتين) وتقسّم الفترات الأطول من ٣٠ يوماً تلقائياً،
  وإعادة المزامنة لا تكرر الفواتير (تحديث بدل إدراج).
- البيانات كلها محلية في `data/pbk.db`. بيانات الاعتماد تُحفظ محلياً بنص صريح —
  لا تشارك هذا المجلد.
- ملفات التصدير تُحفظ أيضاً في `data/exports/`.

## للمطورين

```
app/        FastAPI + SQLite + openpyxl + عميل ETA
static/     واجهة عربية RTL (HTML/CSS/JS بدون إطار عمل)
tests/      pytest — ‎48+ اختباراً (محرك المطابقة، استيراد/تصدير Excel، تكامل API)
```

تشغيل الاختبارات: `python -m pytest tests -q`

مرجع تكامل ETA: <https://sdk.invoicing.eta.gov.eg/api/> — التفاصيل في
`docs/superpowers/specs/2026-06-11-eta-matching-tool-design.md`.
